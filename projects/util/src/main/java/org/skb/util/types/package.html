SKB type library organised in the packages api, atomic and util.

The SKB type library is basically an implementation of the Composite Pattern as a data structure, similar to the way the DEN-ng information model uses this pattern. Similar to the well-known pattern (as defined by the Gang of Four), this implementation composes objects in a tree structure and provides methods to process the resulting tree recursively.

Terminology: The implementation uses the terms api atomic and composite. The api provides interface definitions for atomics and composites. The atomic captures all 'leafs' or 'entries' and provides implementations and wrappers of commonly used other classes, such as the JAVA basic types, ANTLR classes and others.

The sub-packages provide for all types that the SKB system uses.

  • api:
    Contains the interfaces for the type hierarchy (TSBasea, TSAtomic and TSComposite) and some interfaces for specific types (i.e. TSMap, TSTable and TSTableRow).
  • atomic:
    The atomic packages has two sub-packages. First, antlr provides compositions for the ANTLR class Token and the StringTemplate classes StringTemplate and StringTemplate Group. Second, java provides compositions for the basic Java types: Boolean, Byte, Double, Float, Integer, Long, Object, Short and String.
  • util:
    This package contains classes that are very similar to java.utils, such as array lists (for general OatAtomic types and for Strings, map (implementing a linked hash map), PDO (for a simple portable database object), property map (implementing a simple key/value map), table and table row, scope (implementing a utils class for scoped names) and STGManger (providing access to the string template manager of org.skb.utils).
